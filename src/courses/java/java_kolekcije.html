Kolekcije u programskom jeziku Java

Ukoliko broj objekata za rešavanje određenog problema nije unapred poznat, ili postoji mogućnost da će broj objekata biti promenljiv, onda postoji i problem njihovog smeštanja u memoriju. U programskom jeziku Java rešenje tog problema predstavljaju kolekcije - objekti koji se koriste za smeštanje i manipulisanje nedefinisanim brojem objekata. Postoje više različitih tipova kolekcija koje se koriste. Različite vrste kolekcija imaju različite načine manipulisanja podacima koje sadrše. Takođe, različite vrste kolekcija imaju različite nivoe efikasnosti odrađivanja različitih operacija.

Kolekcije u programskom jeziku Java deo su Javinog paketa java.util, pa je za rad sa njima neophdno uvesti ovaj paket. U okviru ovog paketa postoji više vrsta kolekcija:
  • Liste (eng. List) - Lista je kolekcija u kojoj su objekti linearno uređeni, odnosno definsian je redosled objekata kolekcije po kome je određen prvi objekat, drugi objekat, itd. Lista može da sadrži duplirane elemente.
  • Skupovi (eng. Set) - Osnovno svojstvo skupa je da ne postoje duplikati objekata u skupu. U skupu se u opštem slucaju ne podrazumeva da među članovima skupa postoji bilo kakav međusobni poredak.
  • Redovi (eng. Queue) - Redovi se koriste za skladištenje objekata koji se uređuju po principu FIFO - first-in, first-out, odnosno po redosledu ubacivanja i uzimanja elemenata iz kolekcije.

Zajednička svojstva ovih kolekcija se definišu u interfejsima, a njihova implementacija (primena) se realizuje u vidu odgovarajućih klasa prikazanih na slici 3.
