Interfejs SortedSet


Interfejs SortedSet nasleduje interfejs Set i deklariše ponašanje rastuce uredenog skupa.

SortedSet je genericki interfejs koji se deklariše na sledeci nacin:
Interface SortedSet <E> gde E oznacava tip objekta koji ce uredeni skup sadržati.


Pored metoda definisane interfejsom Set, interfejs SortedSet deklariše i sopstvene metode nabrojane u tabeli 5.

Tabela 5. Metode definisane interfesjom SortedSet
Metoda Opis
Comparator comparator() Vraca komparator uredenog skupa. Ako je skup prirodno urešden, vraca se vrednost null.
E first() Vraca prvi element uredenog skupa.
SortedSet headSet(E kraj) Vraca podskup uredenog skupa i to od pocetka skupa do elementa kraj -1.
E last() Vraca poseldnji element uredenog skupa.
SortedSet subSet(E pocetak, E kraj) Vraca podskup uredenog skupa i to od elementa pocetak pa do elementa kraj -1.
SortedSet tailSet(E pocetak) Vraca podskup uredenog skupa i to od elementa pocetak pa do kraja skupa.

Interfejs SortedSet definiše metode koje olakšavaju rad sa skupovima. Prvi element skupa dobija se korišcenjem metoda first(), poslednji metodom last(), dok se podskup uredenog skupa može dobiti korišcenjem metoda subSet(), headSet(), ili tailSet().

Izuzeci koji se generišu prilikom rada sa ovim metodama su:
  1. ClassCastExeption – Ovaj izuzetak se generiše kada su dva objekta medusobno nekompatibilna.
  2. IlegalArgumentExeption – Ovaj izuzetak se generiše ukoliko je metodi prosleden nedozvoljeni argument.
  3. NullPointerExeption – Ovaj izuzetak se generiše pri pokušaju smeštanja null objekata u skup.
  4. NoSuchElementExeption – Ovaj izuzetak se generiše kada u skupu nema elemenata.
