Interfejs SortedSet

Interfejs SortedSet nasleđuje interfejs Set i deklariše ponašanje rastuće uređenog skupa.

SortedSet je generički interfejs koji se deklariše na sledeći način:
Interface SortedSet <E> gde E oznčava tip objekta koji će uređeni skup sadržati.

Pored metoda definisane interfejsom Set, interfejs SortedSet deklariše i sopstvene metode nabrojane u tabeli 5.

Tabela 5. Metode definisane interfesjom SortedSet
Metoda Opis

Comparator <? super E> comparator()

Vraća komparator uređenog skupa. Ako je skup prirodno urešđen, vraća se vrednost null.

E first()

Vraća prvi element uređenog skupa.

SortedSet<E> headSet(E kraj)

Vraća podskup uređenog skupa i to od početka skupa do elementa kraj -1.

E last()

Vraća poseldnji element uređenog skupa.

SortedSet<E> subSet(E početak, E kraj)

Vraća podskup uređenog skupa i to od elementa početak pa do elementa kraj -1.

SortedSet<E> tailSet(E početak)

Vraća podskup uređenog skupa i to od elementa početak pa do kraja skupa.

Interfejs SortedSet definiše metode koje olakšavaju rad sa skupovima. Prvi element skupa dobija se korišćenjem metoda first(), poslednji metodom last(), dok se podskup uređenog skupa može dobiti korišćenjem metoda subSet(), headSet(), ili tailSet().

Izuzeci koji se generišu prilikom rada sa ovim metodama su:
  • ClassCastExeption – Ovaj izuzetak se generiše kada su dva objekta međusobno nekompatibilna
  • IlegalArgumentExeption – Ovaj izuzetak se generiše ukoliko je metodi prosleđen nedozvoljeni argument
  • NullPointerExeption – Ovaj izuzetak se generiše pri pokušaju smeštanja null objekata u skup
  • NoSuchElementExeption – Ovaj izuzetak se generiše kada u skupu nema elemenata
