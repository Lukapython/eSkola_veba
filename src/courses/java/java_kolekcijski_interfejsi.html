Kolekcijski interfejsi

Definisano je više interfejsa za rad sa kolekcijama:
  • Collection interfejs - osnovni interfejs Javinih kolekcija. Implemetiraju ga sve kolekcijske klase. Ovaj interfejs definiše kakav treba da bude interfejs svake kolekcije i time definiše koje sve osnovne funkcije treba da postoje.
  • List interfejs - deklariše ponašanje kolekcije koja se sastoji od sekvence elemenata. Elementi se mogu umetati ili im se može pristupiti na osnovu njihovog indeksa koji počinje od nule. Lista može da sadrži duplirane elemente.
  • Set interfejs - deklariše ponašanje kolekcije koja ne dozvoljava dupliranje elemenata. Veoma je sličan interfejsu Collection.
  • SortedSet interfejs - deklariše ponašanje rastuće uređenog skupa.
  • Queue interfejs - deklariše ponašanje reda čekanja koji se često ponaša kao lista s koje prvo izlazi onaj element koji je na nju prvi postavljen. Elementi se iz reda mogu dohvatiti ili ukloniti samo sa vrha reda. Sa druge strane, dodavanje elementa moguće je samo na kraj reda, pod uslovom da za element ima dovoljno mesta u redu.
  • Deque intrefejs - deklariše ponašanje reda sa dva kraja. Ovaj red funkcioniše kao standardni red (eng. queue) po principu ”prvi koji uđe, prvi izlazi“ (eng. FIFO – first in – first out), ili kao stek, po principu „poslednji koji uđe, prvi izlazi“ (eng LIFO – last in – last out).

Pored ovih interfejsa koriste se i interfejsi Comparator, Iterator i ListIterator. Interfejs Comparator definiše način poređenja dva objekta, dok interfejsi Iterator i ListIterator definišu kako se pojedinačno pristupa objektima u kolekciji.
