Kolekcijski interfejsi

Definisano je više interfejsa za rad sa kolekcijama:

  1. Collection interfejs - osnovni interfejs Javinih kolekcija. Implemetiraju ga sve kolekcijeske klase. Ovaj interfejs definiše kakav treba da bude interfejs svake kolekcije i time definiše koje sve osnovne funkcije treba da postoje.
  2. List interfejs - deklariše ponašanje kolekcije koja se sastoji od sekvence elemenata. Elementi se mogu umetati ili im se može pristupiti na osnovu njihovog indeksa koji pocinje od nule. Lista može da sadrži duplirane elemente.
  3. Set interfejs - deklariše ponašanje kolekcije koja ne dozvoljava dupliranje elemenata. Veoma je slican interfejsu Collection.
  4. SortedSet interfejs - deklariše ponašanje rastuce uredenog skupa.
  5. Queue interfejs - deklariše ponašanje reda cekanja koji se cesto ponaša kao lista s koje prvo izlazi onaj element koji je na nju prvi postavljen. Elementi se iz reda mogu dohvatiti ili ukloniti samo sa vrha reda. Sa druge strane, dodavanje elementa moguce je samo na kraj reda, pod uslovom da za element ima dovoljno mesta u redu.
  6. Deque intrefejs - deklariše ponašanje reda sa dva kraja. Ovaj red funkcioniše kao standardni red (eng. queue) po principu ”prvi koji ude, prvi izlazi“ (eng. FIFO – first in – first out), ili kao stek, po principu „poslednji koji ude, prvi izlazi“ (eng LIFO – last in – last out).

Pored ovih interfejsa koriste se i interfejsi Comparator, Iterator i ListIterator. Interfejs Comparator definiše nacin poredenja dva objekta, dok interfejsi Iterator i ListIterator definišu kako se pojedinacno pristupa objektima u kolekciji.
