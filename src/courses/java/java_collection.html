Interfejs Collection

Interfejs Collection je osnovni interfejs Javinih kolekcija. Ovaj interfejs definiše osnovne metode koje ima svaka kolekcija. Collection je genericki interfejs koji se deklariše na sledeci nacin: Interface Collection <E> gde E oznacava tip objekta koji ce kolekcija sadržati. Pošto sve kolekcije implementiraju ovaj interfejs, poznavanje njegovih metoda je kljucno za razumevanje opštih principa rada sa kolekcijama. Metode definisane interfejsom Collection prikazane su u tabeli 1.


Tabela 1. Metode Collection interfejsa
Metoda Opis
boolean add(E objekat) Dodaje objekat u kolekciju. Ako operacija uspe, vraca vrednost true. U suprotnom vraca false.
boolean addAll(Collection kolekcija) Dodaje sve elemente navedene kolekcije na kraj liste, u poretku kojim ih vraca iterator za kolekciju c. Ako operacija uspe, vraca vrednost true. U suprotnom vraca false.
void clear() Briše sve elemente iz liste.
boolean contains(Object o) Vraca true ako kolekcija sadrži sve elemente kolekcije. U suprotnom vraca vrednost false.
boolen containsAll((Collection kolekcija) Vraca true ako kolekcija sadrži sve elemente kolekcije. U suprotnom vraca vrednost false.
boolean equals(Object obj) Ako su obj i kolekcija jednaki, vraca true, u suprotnom vraca false.
int hasCode() Vraca kljuc za heširanje kolekcija.
boolean isEmpty() Vraca true ako je lista prazna, u suprotnom vraca false.
Iterator iterator() Vraca iterator za pozivajucu kolekciju.
boolean remove(Object o) Briše jednu instancu objekta o iz kolekcije. Ako operacija uspe, vraca vrednost true. U suprotnom vraca false.
boolean removeAll(Collection kolekcija) Uklanja sve objekte iz kolekcije. Vraca true ako je operacija uspela, u suprotnom vraca false.
boolean retainAll(Collection c) Zadržava u kolekciji samo elemente koji su sadržani u navedenoj kolekciji c.
int size() Vraca broj elemenata u kolekciji.
Object[] toArray() Vraca niz koji sadrži sve elemente kolekcije. Elementi niza su kopije elemenata kolekcije.
T[] toArray(T niz[]) Vraca niz koji sadrži kopije samo onih elemenata kolekcije ciji tip odgovara elementima niza. Ako velicina niza odgovara broju elemenata koji se poklapaju, oni se vracaju u niz. Ako je velicina niza manja od broja elemenata koji se poklapaju, vraca se novi niz pocetne velicine. Kada je velicina niza veca od broja elemenata koji se poklapaju, element niza iza poslednjeg clana kolekcije dobija vrednost null. Ukoliko bilo koji element kolekcije ima tip drugaciji od podtipa niz, generiše se izuzetak ArrayStoreExeption()


Dakle, objekat ili grupa objekata može se dodati u kolekciju pomocu metode add() i addAll(), pri cemu se vodi racuna da objekat, odnosno objekti koji se dodaju u kolekciju budu kompatibilani sa tipom podataka koji kolekcija ocekuje. Metode remove() i removeAll() se koriste za brisanje objekta ili grupe objekata iz kolekcije, dok metoda retailAll() uklanja sve elemente izuzev onih iz zadate grupe. Brisanje svih elemenata kolekcije postiže se metodom clear().

Da li kolekcija sadrži odredeni objekat ili sve elemetne druge kolekcije može se utvrditi korišcenjem metode contains() i containsAll(). Da li je kolekcija prazna, utvrduje se metodom isEmpty(), a broj njenih elemenata dobija se pozivom metode size().

Ponekad se sadržaj kolekcije može bolje obraditi pomocu nizova. Preslikavanje izmedu kolekcija i nizova može se ostvariti korišcenjem metoda toArray()i toArray(T niz[]). Prva metoda vraca niz objekata tipa Object, a druga niz elemenata istog tipa T. Druga metoda je cesto pogodnija zato što vraca niz traženog tipa.

Još jedna veoma važna metoda jeste metoda interator() koja vrace iterator za kolekciju. Iteratori se cesto koriste pri radu sa kolekcijama.

Izuzeci koji se generišu prilikom rada sa ovim metodama su:
  1. UnsupportedOperationExeption – Ovaj izuzetak se generiše kada kolekcija ne može da se promeni.
  2. ClassCastExeption – Ovaj izuzetak se generiše kada su dva objekta medusobno nekompatibilna.
  3. IlegalArgumentExeption – Ovaj izuzetak se generiše ukoliko je metodi prosleden nedozvoljeni argument.
  4. NullPointerExeption – Ovaj izuzetak se generiše pri pokušaju smeštanja null objekata u kolekciju u kojoj null elementi nisu dozvoljeni.
  5. IlegalStateExeption - Ovaj izuzetak se generiše pri pokušaju dodavanja novog elementa vec popunjenoj kolekciji - kolekciji fiksne duzine.
