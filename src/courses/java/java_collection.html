Interfejs Collection

Interfejs Collection je osnovni interfejs Javinih kolekcija. Ovaj interfejs definiše osnovne metode koje ima svaka kolekcija. Collection je generički interfejs koji se deklariše na sledeći način: Interface Collection <E> gde E označava tip objekta koji ce kolekcija sadržati.
Pošto sve kolekcije implementiraju ovaj interfejs, poznavanje njegovih metoda je ključno za razumevanje opštih principa rada sa kolekcijama. Metode definisane interfejsom Collection prikazane su u tabeli 1.

Tabela 1. Metode Collection interfejsa
Metoda Opis

boolean add(E objekat)

Dodaje objekat u kolekciju. Ako operacija uspe, vraća vrednost true. U suprotnom vraća false.

boolean addAll(Collection<? extends E>  kolekcija)

Dodaje sve elemente navedene kolekcije na kraj liste, u poretku kojim ih vraća iterator za kolekciju c.

Ako operacija uspe, vraća vrednost true . U suprotnom vraća false .

void clear()          

Briše sve elemente iz liste.

boolean contains(Object o)

Vraća true ako lista sadži navedeni element.

boolen containsAll(Collection<?> kolekcija)

Vraća true ako kolekcija sadrži sve elemente kolekcije. U suprotnom vraća vrednost false.

boolean equals(Object obj)

Ako su obj i kolekcija jednaki, vraća true, u suprotnom vraća  false.

int hasCode()                 

Vraća ključ za heširanje kolekcija.

boolean isEmpty()

Vraća true ako je lista prazna, u suprotnom vraća false.

Iterator<E> iterator()        

Vraća iterator za pozivajuću kolekciju.

boolean remove(Object o)

Briše jednu instancu objekta o iz kolekcije. Ako operacija uspe, vraća vrednost true. U suprotnom vraća false

boolean removeAll(Collection <?> kolekcija)

Uklanja sve objekte iz kolekcije. Vraća true ako je operacija uspela, u suprotnom vraća false.

boolean retainAll(Collection<?> c)

Zadržava u kolekciji samo elemente koji su sadržani u navedenoj kolekciji c.

int size()             

Vraća broj elemenata u kolekciji.

Object[] toArray()             

Vraća niz koji sadrži sve elemente kolekcije. Elementi niza su kopije elemenata kolekcije.

<T>T[] toArray(T niz[])

Vraća niz koji sadrži kopije samo onih elemenata kolekcije čiji tip odgovara elementima niza. Ako veličina niza odgovara broju elemenata koji se poklapaju, oni se vraćaju u niz. Ako je veličina niza manja od broja elemenata koji se poklapaju, vraća se novi niz početne veličine. Kada je veličina niza veća od broja elemenata koji se poklapaju, element niza iza poslednjeg člana kolekcije dobija vrednost null. Ukoliko bilo koji element kolekcije ima tip drugačiji od podtipa niz, generiše se izuzetak  ArrayStoreExeption()

Dakle, objekat ili grupa objekata može se dodati u kolekciju pomoću metode add() i addAll(), pri čemu se vodi računa da objekat, odnosno objekti koji se dodaju u kolekciju budu kompatibilani sa tipom podataka koji kolekcija očekuje. Metode remove() i removeAll() se koriste za brisanje objekta ili grupe objekata iz kolekcije, dok metoda retailAll() uklanja sve elemente izuzev onih iz zadate grupe. Brisanje svih elemenata kolekcije postiže se metodom clear().

Da li kolekcija sadrži određeni objekat ili sve elemetne druge kolekcije može se utvrditi korišćenjem metode contains() i containsAll(). Da li je kolekcija prazna, utvrđuje se metodom isEmpty(), a broj njenih elemenata dobija se pozivom metode size().

Ponekad se sadržaj kolekcije može bolje obraditi pomoću nizova. Preslikavanje izmedu kolekcija i nizova može se ostvariti korišćenjem metoda toArray()i toArray(T niz[]). Prva metoda vraća niz objekata tipa Object, a druga niz elemenata istog tipa T. Druga metoda je često pogodnija zato što vraća niz traženog tipa.

Još jedna veoma važna metoda jeste metoda interator() koja vraća iterator za kolekciju. Iteratori se često koriste pri radu sa kolekcijama.

Izuzeci koji se generišu prilikom rada sa ovim metodama su:
  • UnsupportedOperationExeption – Ovaj izuzetak se generiše kada kolekcija ne može da se promeni
  • ClassCastExeption – Ovaj izuzetak se generiše kada su dva objekta međusobno nekompatibilna
  • IlegalArgumentExeption – Ovaj izuzetak se generiše ukoliko je metodi prosleđen nedozvoljeni argument
  • NullPointerExeption – Ovaj izuzetak se generiše pri pokušaju smeštanja null objekata u kolekciju u kojoj null elementi nisu dozvoljeni
  • IlegalStateExeption - Ovaj izuzetak se generiše pri pokušaju dodavanja novog elementa već popunjenoj kolekciji - kolekciji fiksne dužine
