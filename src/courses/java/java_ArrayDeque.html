Klasa ArrayDeque

Klasa ArrayDeque predstavlja dinimički niz neograničenog kapaciteta, odnosno red. Ovaj red funkcioniše kao standardni red (eng. queue) po principu ”prvi koji uđe, prvi izlazi“ (eng. FIFO – first in – first out), ili kao stek, po principu „poslednji koji uđe, prvi izlazi“ (eng LIFO – last in – last out). Dodavanje elemenata u sredinu nije podržano.

Klasa ArrayDeque je generička klasa koja se definiše na sledeći način:
Class ArrayDeque <E> U ovoj deklaraciji E je tip objekata koji se skaldišti.
Klasa ArrayDeque realizuje interfejse Collection, Iterable i Deque, što znači da klasa ArrayDeque može koristiti sve metode definisane ovim interfejsima.
Konstruktori koje sadži klasa ArrayDeque prikazani su u tabeli 14.

Tabela 14. ArrayDeque konstruktori

Da bi objekat tipa ArrayDeque bio kreiran, neophodno je prvo kreirati njegovu instancu pomoću konstruktora iz tabele 13. U svim navedenim slučajevima, kapacitet ArrayDeque objekata se povećava kada se red popuni elementima.

Primer 20:Upotreba ArrayDeque klase
import java.util.*; public class PrimerZaArrayDeque { public static void main(String[] args) { // Kreiranje objekta tipa ArrayDeque ArrayDeque<String> adg = new ArrayDeque<String>(); // Objekat tipa ArrayDeque koristimo kao stek – ilustracija metoda push() adg.push("A"); adg.push("B"); adg.push("F"); adg.push("G"); System.out.print("Praznjenje steka: "); // Praznjenje steka - ilustracija metoda peek() i pop() while(adg.peek()!= null){ System.out.print(adg.pop() + " "); System.out.print(" "); } } }
Rezultat programa:
Praznjenje steka: G F B A U ovom primeru klasa ArrayDeque je upotrebljena za izradu steka. Kao što je prikazano u rezultatu programa, element koj je poslednji ušao u red, prvi je izašao iz njega. U ovom primeru korišćene su metode Deque instefejsa.

Važne osobine ArrayDeque klase:
  • Za razliku od običnog reda, moguće je dodavati i brisati elemente sa oba kraja reda u kolekciji tipa ArrayDeque
  • ArrayDeque je red neograničenog kapaciteta
  • ArrayDeque je brži od Steka i LinkedList kolekcije
  • Null objekti su dopušteni
