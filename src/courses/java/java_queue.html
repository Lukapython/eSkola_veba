Interfejs Queue


Red (eng. queue) je kolekcija u kojoj se elementi cuvaju pre obrade. Elementi se iz reda mogu dohvatiti ili ukloniti samo sa vrha reda. To znaci, da nije moguce preskociti pocetni element i dohvatiti ili ukloniti neki drugi element iz reda. Sa druge strane, dodavanje elementa moguce je samo na kraj reda, pod uslovom da za element ima dovoljno mesta u redu.

Queue je genericki interfejs koji se deklariše na sledeci nacin:
Interface Queue<E> gde E oznacava tip objekta koji ce red sadržati.
Ovaj interfejs nasleduje interfejs Collection i dodaje metode definisane u tabeli 6.

Tabela 6. Metode definisane interfesjom Queue
Metoda Opis
E element() Vraca element koji se nalazi na vrhu reda, pri cemu se taj element se ne uklanja. Ako je red na cekanju prazan, generiše se izuzetak NoSuchElementExeption.
boolean offfer(E obj) Pokušava da doda objekat obj u red i vraca true ukoliko je operacija uspela, u suprotnom, vraca false.
E peek() Vraca element koji se nalazi na vrhu reda. Ako je red prazan, vraca null. Element se ne uklanja.
E poll() Vraca element koji se nalazi na vrhu reda, uklanjajuci pri tom taj element. Ako je red prazan, vraca null.
E remove() Uklanja element koji se nalazi na vrhu reda i pri tom ga vraca kao rezultat metode. Ako je red prazan, generiše se izuzetak NoSuchElementExeption.

Postoje dve metode koje ucitavaju i vade element iz reda – poll() i remove(). Razlika izmedu ovih metoda je u tome što metoda poll() vraca null ako je red prazan, a metoda remove() generiše izuzetak NoSuchElementExeption. Takode, postoje i dve metode koje ucitavaju ali ne vade element iz reda – peek() i element(). Kao i kod prethodnih metoda, prva metoda vraca null ako je red prazan, dok druga metoda generiše izuzetak. Kako postoje redovi fiksne dužine koji se mogu napuniti, metoda offer() pokušava da doda element u red. Dakle, ukoliko je red pun, ova metoda ce vratiti vrednost false.

Izuzeci koji se generišu prilikom rada sa ovim metodama su:
  1. ClassCastExeption – Ovaj izuzetak se generiše kada su dva objekta medusobno nekompatibilna.
  2. IlegalArgumentExeption – Ovaj izuzetak se generiše ukoliko je metodi prosleden nedozvoljeni argument.
  3. NullPointerExeption – Ovaj izuzetak se generiše pri pokušaju smeštanja null objekata u red.
  4. NoSuchElementExeption – Ovaj izuzetak se generiše kada u redu nema elemenata.
