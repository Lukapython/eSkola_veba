Interfejs Deque


Interfejs Deque deklariše ponašanje reda sa dva kraja. Ovaj red funkcioniše kao standardni red (eng. queue) po principu ”prvi koji ude, prvi izlazi“ (eng. FIFO – first in – first out), ili kao stek, po principu „poslednji koji ude, prvi izlazi“ (eng LIFO – last in – last out).

Deque je genericki interfejs koji se deklariše na sledeci nacin:
Interface Deque<E> gde E oznacava tip objekta koji ce red tipa Deque sadržati.
Ovaj interfejs nasleduje interfejs Queue i dodaje metode definisane u tabeli 7.

Tabela 7. Metode definisane interfesjom Deque
Metoda Opis
void addFirst(E obj) Smešta obj na pocetak reda tipa Deque. Generiše izuzetak IlegalStateExeption ako više nema mesta u redu ogranicenog kapaciteta.
void addLast(E obj) Smešta obj na kraj reda. Generiše izuzetak IlegalStateExeption ako više nema mesta u redu ogranicenog kapaciteta.
Iterator descending Iterator() Vraca iterator koji pomera s kraja reda na pocetak. Drugim recima vraca obrnuti iterator.
E getFirst() Vraca prvi element u redu. Taj objekat se ne uklanja iz reda. Metoda generiše izuzetak NoSuchElementExeption ako je red prazan.
E getLast() Vraca poslednji element u redu. Taj objekat se ne uklanja iz reda. Metoda generiše izuzetak NoSuchElementExeption ako je red prazan.
boolean offerFirst(E obj) Pokušava da doda obj na pocetak reda. Metoda vraca true ako uspe da doda obj u red, u suprotnom vraca false.
boolean offerLast(E obj) Pokušava da doda obj na kraj reda. Metoda vraca true ako uspe da doda obj u red, u suprotnom vraca false.
E peekFirst() Vraca element koji se nalazi na pocetku reda, odnosno null ako je red prazan. Objekat se ne uklanja iz reda.
E peekLast() Vraca element koji se nalazi na kraju reda, odnosno null ako je red prazan. Objekat se ne uklanja iz reda.
E pollFirst() Vraca element koji se nalazi na pocetku reda, uklanjajuci ga iz reda. Metoda vraca null ako je red prazan.
E pollLast() Vraca element koji se nalazi na kraju reda, uklanjajuci ga iz reda. Metoda vraca null ako je red prazan.
E pop() Vraca element koji se nalazi na pocetku reda, uklanjajuci ga iz reda. Metoda generiše izuzetak NoSuchElementExeption ako je red prazan.
void push(E obj) Postavlja obj na pocetak reda. Generiše izuzetak IlegalStateExeption ako više nema mesta u redu koji je ogranicenog kapaciteta.
E removeFirst() Vraca element koji se nalazi na pocetku reda, uklanjajuci ga iz reda. Metoda generiše izuzetak NoSuchElementExeption ako je red prazan.
boolean removeFirstOccurrence(Object obj) Uklanja prvi primerak objekta obj iz reda. Metoda vraca true ako se izvrši uspesno, odnosno false ako red ne sadrži objekat obj.
E removeLast() Vraca element koji se nalazi na kraju reda, uklanjajuci ga iz reda. Metoda generiše izuzetak NoSuchElementExeption ako je red prazan.
boolean removeLastOccurrence(Object obj) Uklanja poslednji primerak objekta obj iz reda. Metoda vraca true ako se izvrši uspesno, odnosno false ako red ne sadrži objekat obj.

Klasa koja implementira interfejs Deque može imati ogranicen kapacitet, odnosno u takav red se može dodati samo ogranicen broj elemenata. U slucaju da je red popunjem, svaki pokušaj dodavanja novog elementa zavrsice se neuspehom. Postoje po dve metode koje dodaju element na pocetak i kraj reda. Razlika je u tome što metode addFirst() i addLast() generišu izuzetak IlegalStateExeption() ako je red sa ogranicenim kapacitetom popunjen, dok metode offreFirst(), i offerLast() vracaju false ako se u red ne može dodati novi element. Metoda descendingIterator() vraca iterator koji omogucava pristupanje elementima reda obrnutim redosledom. Takode, ovaj interfejs sadrži metode push() i pop(), što omogucava da klase koje implementiraju interfejs Deque rade kao stek.

Izuzeci koji se generišu prilikom rada sa ovim metodama su:
  1. ClassCastExeption – Ovaj izuzetak se generiše kada su dva objekta medusobno nekompatibilna.
  2. IlegalArgumentExeption – Ovaj izuzetak se generiše ukoliko je metodi prosleden nedozvoljeni argument.
  3. NullPointerExeption – Ovaj izuzetak se generiše pri pokušaju smeštanja null objekata u red.
  4. NoSuchElementExeption – Ovaj izuzetak se generiše kada u redu nema elemenata.
  5. IlegalStateExeption - Ovaj izuzetak se generiše pri pokušsaju dodavanja novog elementa u red fiksne dužine koji je popunjen.
