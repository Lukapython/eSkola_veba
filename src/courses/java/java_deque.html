Interfejs Deque

Interfejs Deque deklariše ponašanje reda sa dva kraja. Ovaj red funkcioniše kao standardni red (eng. queue) po principu ”prvi koji uđe, prvi izlazi“ (eng. FIFO – first in – first out), ili kao stek, po principu „poslednji koji uđe, prvi izlazi“ (eng LIFO – last in – last out).

Deque je generički interfejs koji se deklariše na sledeći nacin:
Interface Deque<E> gde E označava tip objekta koji će red tipa Deque sadržati.
Ovaj interfejs nasleđuje interfejs Queue i dodaje metode definisane u tabeli 7.

Tabela 7. Metode definisane interfesjom Deque
Metoda Opis
void addFirst(E obj) Smešta obj na početak reda tipa Deque. Generiše izuzetak IlegalStateExeption ako više nema mesta u redu ograničenog kapaciteta.
void addLast(E obj) Smešta obj na kraj reda. Generiše izuzetak IlegalStateExeption ako više nema mesta u redu ograničenog kapaciteta.
Iterator descending Iterator() Vraća iterator koji pomera s kraja reda na početak. Drugim recima vraća obrnuti iterator.
E getFirst() Vraća prvi element u redu. Taj objekat se ne uklanja iz reda. Metoda generiše izuzetak NoSuchElementExeption ako je red prazan.
E getLast() Vraća poslednji element u redu. Taj objekat se ne uklanja iz reda. Metoda generiše izuzetak NoSuchElementExeption ako je red prazan.
boolean offerFirst(E obj) Pokušava da doda obj na početak reda. Metoda vraća true ako uspe da doda obj u red, u suprotnom vraća false.
boolean offerLast(E obj) Pokušava da doda obj na kraj reda. Metoda vraća true ako uspe da doda obj u red, u suprotnom vraća false.
E peekFirst() Vraća element koji se nalazi na početku reda, odnosno null ako je red prazan. Objekat se ne uklanja iz reda.
E peekLast() Vraća element koji se nalazi na kraju reda, odnosno null ako je red prazan. Objekat se ne uklanja iz reda.
E pollFirst() Vraća element koji se nalazi na pocetku reda, uklanjajuci ga iz reda. Metoda vraća null ako je red prazan.
E pollLast() Vraća element koji se nalazi na kraju reda, uklanjajući ga iz reda. Metoda vraća null ako je red prazan.
E pop() Vraća element koji se nalazi na pocetku reda, uklanjajući ga iz reda. Metoda generiše izuzetak NoSuchElementExeption ako je red prazan.
void push(E obj) Postavlja obj na pocetak reda. Generiše izuzetak IlegalStateExeption ako više nema mesta u redu koji je ograničenog kapaciteta.
E removeFirst() Vraca element koji se nalazi na početku reda, uklanjajući ga iz reda. Metoda generiše izuzetak NoSuchElementExeption ako je red prazan.
boolean removeFirstOccurrence(Object obj) Uklanja prvi primerak objekta obj iz reda. Metoda vraća true ako se izvrši uspesno, odnosno false ako red ne sadrži objekat obj.
E removeLast() Vraća element koji se nalazi na kraju reda, uklanjajući ga iz reda. Metoda generiše izuzetak NoSuchElementExeption ako je red prazan.
boolean removeLastOccurrence(Object obj) Uklanja poslednji primerak objekta obj iz reda. Metoda vraća true ako se izvrši uspesno, odnosno false ako red ne sadrži objekat obj.

Klasa koja implementira interfejs Deque može imati ograničen kapacitet, odnosno u takav red se može dodati samo ograničen broj elemenata. U slučaju da je red popunjem, svaki pokušaj dodavanja novog elementa zavrsiće se neuspehom. Postoje po dve metode koje dodaju element na početak i kraj reda. Razlika je u tome što metode addFirst() i addLast() generišu izuzetak IlegalStateExeption() ako je red sa ograničenim kapacitetom popunjen, dok metode offreFirst(), i offerLast() vraćaju false ako se u red ne može dodati novi element. Metoda descendingIterator() vraća iterator koji omogucava pristupanje elementima reda obrnutim redosledom. Takođe, ovaj interfejs sadrži metode push() i pop(), što omogućava da klase koje implementiraju interfejs Deque rade kao stek.

Izuzeci koji se generišu prilikom rada sa ovim metodama su:
  • ClassCastExeption – Ovaj izuzetak se generiše kada su dva objekta medusobno nekompatibilna
  • IlegalArgumentExeption – Ovaj izuzetak se generiše ukoliko je metodi prosleđen nedozvoljeni argument
  • NullPointerExeption – Ovaj izuzetak se generiše pri pokušaju smeštanja null objekata u red
  • NoSuchElementExeption – Ovaj izuzetak se generiše kada u redu nema elemenata
  • IlegalStateExeption - Ovaj izuzetak se generiše pri pokušaju dodavanja novog elementa u red fiksne dužine koji je popunjen
