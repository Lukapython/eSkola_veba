Interfejs List


Interfejs List proširuje interfejs Collection, i deklariše ponašanje kolekcije koja sadrži sekvencu elemenata. Elementi se mogu umetati ili im se može pristupati na osnovu njhovog položaju u listi, pomocu indeksa cija vrednost pocinje od nule. Lista može da sadrži duplikate elemenata.

List je genericki interfejs koji se deklariše na sledeci nacin:
Interface List <E> gde E oznacava tip objekta koji ce kolekcija sadržati.


Osim metoda definisanih interfejsom Collection prikazanih u tabeli 1, interfejs List definiše i svoje metode prikazane u tabeli 4.

Tabela 4. Metode definisane interfejsom List
Metoda Opis
void add(int indeks, E obj) Na mesto zadato indeksom, umece navedeni objekat. Umetanjem se ne gubi nijedan element.
boolean addAll(int indeks, Collection kolekcija) Umece sve elemente kolekcije u listu na mesto zadato indeksom. Umetanjem se ne gubi nijedan element. Ukoliko se umetanje uspešno obavi, vraca true, u suprotnom vraca false.
E get(int indeks) Vraca objekat zadatog indeksa sa liste.
int indexOf(Object obj) Vraca indeks prve pojave navedenog objekta u kolekciji, ili -1 ukoliko lista ne sadži navedeni objekat.
int lastIndexOf(Object obj) Vraca indeks poslednje pojave navedenog objekta u listi, ili -1 ukoliko lista ne sadži navedeni objekat.
ListIterator listIterator() Vraca iterator liste.
ListIterator listIterator(int indeks) Vraca iterator liste koji pocinje od mesta zadatog parametrom indeks.
E remove(int indeks) Iz liste briše element zadatog indeksa. Metoda vraca uklonjeni element.
E remove(int indeks) Iz liste briše element zadatog indeksa. Metoda vraca uklonjeni element.
E set(int indeks, E obj) Postavlja obj u listu na mesto zadato indeksom.
List subList(int fromIndex, int toIndex) Vraca podlistu koja obuhvata deo liste izmedu zadatih pozicija, ukljucujuci prvu i ne ukljucujuci drugu poziciju

Kao što je prikazano u tabeli 4, verzijama metoda add() i addAll() definisanim interfejsom Collection, interfejs List dodaje metode add(int, E) i addAll(int, Collection) kojima se elementi unose na mesto zadato indeksom. Takode, interfejs List menja semantiku metoda add(int, E) i addAll(int, Collection) definisanih u interfesju Collection tako da one dodaju elemente na kraj liste.
Do objekta smeštenog na odredenoj lokaciji može se doci metodom get(), dok se indeks odredenog objekta može dobiti pomocu metode indexOf(), ili lastIndexOf(). Kada je indeks odredenog objekta poznat, može se metodom set() postaviti odredena vrednost na indeksom zadatoj lokaciji. Takode, podlista odredene liste može se dobiti metodom sublist(), navodenjem pocetnog i krajnjeg indeksa podliste.

Izuzeci koji se generišu prilikom rada sa ovim metodama su:
  1. UnsupportedOperationExeption – Ovaj izuzetak se generiše kada kolekcija ne može da se promeni.
  2. ClassCastExeption – Ovaj izuzetak se generiše kada su dva objekta medusobno nekompatibilna.
  3. IlegalArgumentExeption – Ovaj izuzetak se generiše ukoliko je metodi prosleden nedozvoljeni argument.
  4. NullPointerExeption – Ovaj izuzetak se generiše pri pokušaju smeštanja null objekata listi.
  5. IndexOutOfBoundsExeption - Ovaj izuzetak se generiše ukoliko se upotrebi indeks neispravne vrednosti.
