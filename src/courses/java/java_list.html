Interfejs List

Interfejs List proširuje interfejs Collection, i deklariše ponašanje kolekcije koja sadrži sekvencu elemenata. Elementi se mogu umetati ili im se može pristupati na osnovu njhovog položaja u listi, pomoću indeksa čija vrednost počinje od nule. Lista može da sadrži duplikate elemenata.

List je genericki interfejs koji se deklariše na sledeci nacin:
Interface List <E> gde E označava tip objekta koji će kolekcija sadržati.
Osim metoda definisanih interfejsom Collection prikazanih u tabeli 1, interfejs List definiše i svoje metode prikazane u tabeli 4.

Tabela 4. Metode definisane interfejsom List
Metoda Opis

void add(int indeks, E obj)

Na mesto zadato indeksom, umeće navedeni objekat. Umetanjem se ne gubi nijedan element.

boolean addAll(int indeks, Collection<? extends E> kolekcija)

Umeće sve elemente kolekcije u listu na mesto zadato indeksom. Umetanjem se ne gubi nijedan element. Ukoliko se umetanje uspešno obavi, vraća true, u suprotnom vraća false.

E get(int indeks)

Vraća objekat zadatog indeksa sa liste.

int indexOf(Object obj)

Vraća indeks prve pojave navedenog objekta u kolekciji, ili -1  ukoliko lista ne sadži navedeni objekat.

int lastIndexOf(Object obj)

Vraća indeks poslednje pojave navedenog objekta u listi, ili -1  ukoliko lista ne sadži navedeni objekat.

ListIterator<E>  listIterator()

Vraća iterator liste.

ListIterator<E> listIterator(int indeks)

Vraća iterator liste koji počinje od mesta zadatog parametrom indeks.

E remove(int indeks)

Iz liste briše element zadatog indeksa. Metoda vraća uklonjeni element.

E set(int indeks, E obj)

Postavlja obj u listu na mesto zadato indeksom.

List<E> subList(int fromIndex, int toIndex)

 

 

 

Vraća podlistu koja obuhvata deo liste između zadatih pozicija, uključujući prvu i ne uključujući drugu poziciju.

Kao što je prikazano u tabeli 4, verzijama metoda add() i addAll() definisanim interfejsom Collection, interfejs List dodaje metode add(int, E) i addAll(int, Collection) kojima se elementi unose na mesto zadato indeksom. Takođe, interfejs List menja semantiku metoda add(int, E) i addAll(int, Collection) definisanih u interfesju Collection tako da one dodaju elemente na kraj liste.
Do objekta smeštenog na određenoj lokaciji može se doći metodom get(), dok se indeks određenog objekta može dobiti pomoću metode indexOf(), ili lastIndexOf(). Kada je indeks određenog objekta poznat, može se metodom set() postaviti određena vrednost na indeksom zadatoj lokaciji. Takođe, podlista određene liste može se dobiti metodom sublist(), navođenjem početnog i krajnjeg indeksa podliste.

Izuzeci koji se generišu prilikom rada sa ovim metodama su:
  • UnsupportedOperationExeption – Ovaj izuzetak se generiše kada kolekcija ne može da se promeni
  • ClassCastExeption – Ovaj izuzetak se generiše kada su dva objekta medusobno nekompatibilna
  • IlegalArgumentExeption – Ovaj izuzetak se generiše ukoliko je metodi prosleđen nedozvoljeni argument
  • NullPointerExeption – Ovaj izuzetak se generiše pri pokušaju smeštanja null objekata listi
  • IndexOutOfBoundsExeption - Ovaj izuzetak se generiše ukoliko se upotrebi indeks neispravne vrednosti
