Radni okvir Spring Framework

Radni Okvir Spring Framework predstavlja osnovu familije radnih okvira Spring. Ovaj radni okvir sadrži osnovne biblioteke na kojima se zasnivaju svi drugi radni okviri. Najvažnija osobina koju pruža jeste podrška za injektovanje zavisnosti preko IoC (Inversion of Control) kontejnera i aspektno orijentisano programiranje.

Principi radnog okvira Spring

Korisno je proučiti principe na kojima se ovaj radni okvir zasniva.

  • Spring pruža mogućnost donošenja odluka vezanih za dizajn veoma kasno u razvoju aplikacije. Klasičan primer predstavlja mogućnost promene provajdera perzistencije bez promene koda (na primer prelazak sa MySQL na PostgreSQL RDMBS).
  • Spring promoviše fleksibilnost tako što podržava razvoj aplikacija širokog opsega primena na više različitih načina.
  • Spring je razvijan tako da se sa novom verzijom uvede što manje breaking promena, tako da je kompatibilnost sa starijim verzijama na veoma visokom nivou.
  • Aplikacioni programski interfejs radnog okvira Spring Framework je pažljivo dizajniran kako bi bio što intutivniji i održao se bez promena kroz što više verzija.
  • Spring je jedan od retkih radnih okvira koji ima veoma čistu strukturu koda i nema cirkularnih zavisnosti između paketa.

Spring Framework

Radni okvir Spring Framework može se podeliti u nekoliko celina.

  • Jezgro - U jezgru radnog okvira nalazi se podrška za injekvovanje zavisnosti (tj. inverziju kontrole), aspektono orijentisano programiranje, obradu događaja, internacionalizaciju, validaciju, konverziju tipova, kao i SpEL (Spring Expression Language).
  • Test - Radni okvir Spring Framework znatno olakšava testiranje aplikacija prižajući biblioteku za mokovanje objekata, radne okvire TestContext i Spring MVC Test, kao i biblioteku WebTestClient.
  • Pristup podacima - Spring Framework sadrži veoma opširnu biblioteku za olakšavanje pristupa podacima i kominukacije između biznis sloja i sloja perzistencije. Biblioteka sadrži podršku za transkakcije, DAO (Data Access Object), JDBC, objektno-relaciono mapiranje, ali i maršalovanje fajlova u XML formatu.
  • Veb servleti - Radni okvir Spring framework u sebi sadrži radni okvir za razvoj veb aplikacija pod nazivom Spring MVC, a takođe pruža podršku za protokole kao što su WebSocket, SockJS i STOMP.
  • Reaktivno veb programiranje - Za podršku reaktivnom programiranje Spring Framework obezbeđuje radni okvir Spring WebFlux, reaktivnog klijenta pod nazivom WebClient kao i podršku za WebSocket protokol.
  • Integracija - Deo radnog okvira Spring Framework čini i biblioteka za podršku integraciji poslovnih aplikacija pomoću različitih Java tehnologija u ovom domenu kao što su JMS (Java Messaging System), JCA (Java EE Connector Architecture), JMX (Java Management Extensions) itd.
  • Programski jezici - Iako nastao kao Java radni okvir, Spring Framework danas podržava korišćenje većeg broja programskih jezika iz Java porodice: Kotlin, Groovy, kao i dinamičke jezike JRuby i BeanShell.
