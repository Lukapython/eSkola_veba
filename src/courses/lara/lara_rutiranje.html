Rutiranje


Gde se zapravo nalazi stranica koja se otvara sa http://127.0.0.1:8000 ? Izgled stranice nalazi se u folderu views, koji se nalazi u folderu resources. Welcome.blade.php fajl predstavlja izgled stranice koja se otvara sa http://127.0.0.1:8000 . Otvorite sada folder rotes i web.php fajl! Ovde se definišu putanje za veb aplikaciju. Sa return view('welcome') dobija se izgled stranice welcome iz welcome.blade.php. Može se pozvati neka druga stranica iz resorces/views, na primer stranica about.php koja se pravi u folderu views. Kasnije će biti napravljena stranica about.blade.php koristeći jednostavan html kod. Ako se sada u fajlu routes/web.php kod Route::get('/', function () { return view('welcome'); }); zameni kodom Route::get('/', function () { echo "Hello World"; }); nakon što se osveži stranica, dobiće se ispis Hello World. Šta se dešava ako se nakon porta 8000 doda jos nešto? Na primer, napraviti view koji će se prikazati nakon http://127.0.0.1:8000/about zahteva. U routes/web.php fajl treba dodati sledeći deo koda: Route::get('about', function () { return view('about')"; }); Dakle, otvoriće se stranica about.blade.php koja se pravi i čuva u resources/views folderu. Ona će imati jednostavan html kod. npr. <!DOCTYPE html> <html> <head> <title></title> </head> <body> About Us </body> </html> Nakon što se pokrene ova jednostavna aplikacija, dobija se sledeće:

Naravno, mogu se preneti još neki parametri koji ne moraju biti fiksni. Parametar naslov se prosleđuje funkciji. Aplikacija se može pokrenuti i dobija se sledeće: Primetite, aplikacija je pokrenuta sa http://127.0.0.1:8000/pobeda-kosarkasa Sada naslov može biti nešto drugo i dobija se: Upravo tu promenu putanje, koja se dozvoljava bez ekplicitno naglašene promene putanje u kodu, omogućava parametarsko rutiranje.
