XML Schema

XML Schema je alternativni način za specifikovanje strukture XML dokumenta.
Ona je u prednosti u odnosu na DTD iz nekoliko razloga:
  • XML Schema je pisana u XML-u.
  • XML Schema je proširiva.
  • XML Schema podržava različite tipove podataka.
  • XML Schema podržava namespace.
Za primer koji smo već imali "obavestenje.xml": <?xml version="1.0" encoding="UTF-8"?> <obavestenje> <od>Autor</od> <za>Čitalac</za> <naslov>Obaveštenje</naslov> <poruka> Počinje kurs. Srećno! </poruka> </obavestenje> XML Schema "obavestenje.xsd"može izgledati otprilike ovako: <?xml version="1.0"?> <xs:schema xmlns:xs="URI"> <xs:element name="obavestenje"> <xs:complexType> <xs:sequence> <xs:element name="od" type="xs:string"/> <xs:element name="za" type="xs:string"/> <xs:element name="naslov" type="xs:string"/> <xs:element name="poruka" type="xs:string"/> </xs:sequence> </xs:complexType> </xs:element> </xs:schema> Ona pokazuje sledeće:
  • služi za definisanje namespacea za prefiks xs.
  • pokazuje da je element "obavestenje" složen, odnosno da sadrži druge elemente.
  • pokazuje da sledi niz elemenata koji pripadaju "obavestenju".
  • pokazuje da je element "od" tipa string. Na isti način, pokazano je i da su elementi "za", "naslov" i "poruka" takođe tipa string.


Referenca ka ovoj schemi se u XML dokument unosi kao atribut elementa "obavestenje", na sledeći način: <?xml version="1.0"?> <obavestenje xmlns:xsi="URI" xsi:schemaLocation="URI obavestenje.xsd"> <od>Autor</od> <za>Čitalac</za> <naslov>Obaveštenje</naslov> <poruka> Počinje kurs. Srećno! </poruka> </obavestenje> Pri čemu se mora definisati namespace za prefiks xsi (XML Schema Instance), a tek onda iskoristiti atribut za lokaciju dokumenta xsi:schemaLocation="URI obavestenje.xsd" Ovaj atribut zapravo ima dve vrednosti razdvojene "space"-om (prazninom). Prva vrednost je namespace koji se koristi, a druga lokacija XML Schema dokumenta.

Prednosti korišćenja XML Scheme

Velika prednost XML Scheme je to što podržava različite tipove podataka, pa je tako lakše opisati sadržaj dokumenta, proveriti ispravnost podataka i konvertovati tip podatka u neki drugi. Takođe, značajno je to što je ona napisana u XML-u. Prema tome, ne mora se učiti novi jezik, može se koristiti isti editor kao i za XML za uredjivanje XML Schema dokumenata, isti parser za parsiranje XML Schema dokumenta, mogu se transformisati XSLT-om.
