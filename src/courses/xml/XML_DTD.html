XML DTD

XML DTD (Document Type Definition) je originalan način za specifikovanje strukture XML dokumenta. Zapravo, on specifikuje gradivne blokove i redosled elemenata u XML dokumentu. XML DTD ima potpuno drugačiju sintaksu od XML-a. XML DTD može biti deklarisan unutar XML dokumenta ili kao spoljašnja referenca. Pogledajmo sledeći primer: <?xml version="1.0" encoding="UTF-8"?> <!DOCTYPE obavestenje SYSTEM "Obaveštenje.dtd"> <obavestenje> <od>Autor</od> <za>Čitalac</za> <naslov>Obaveštenje</naslov> <poruka> Počinje kurs. Srećno! </poruka> </obavestenje> Ovde je DOCTYPE deklaracija zapravo referenca na spoljašnji DTD dokument "Obaveštenje.dtd", a izgled tog DTD dokumenta je: <!DOCTYPE obavestenje [ <!ELEMENT obavestenje (od, za, naslov, poruka)> <!ELEMENT od (#PCDATA)> <!ELEMENT za (#PCDATA)> <!ELEMENT naslov (#PCDATA)> <!ELEMENT poruka (#PCDATA)> ]> Naveden primer DTD dokumenta definiše sledeće:
  • !DOCTYPE obavestenje definiše da je koreni element odgovarajućeg XML dokumenta obavestenje.
  • !ELEMENT obavestenje definiše da element obavestenje sadrži četiri elementa (od, za, naslov i poruka).
  • !ELEMENT od definiše da je element od tipa "#PCDATA" (Parsed Character DATA - tekst izmedju otvorene i zatvorene etikete XML elementa koji se analizira od strane parsera i koju je pre svega moguće parsirati).
  • !ELEMENT za definiše da je element za tipa "#PCDATA"
  • !ELEMENT naslov definiše da je element naslov tipa "#PCDATA"
  • !ELEMENT poruka definiše da je element poruka tipa "#PCDATA"
XML DTD se može koristiti i za deklaraciju entiteta, odnosno definisanje specijalnih karaktera i stringova koji se koriste u dokumentu. Primer: <?xml version="1.0" encoding="UTF-8"?> <!DOCTYPE pismo [ <!ENTITY autor "Autor: Pera Perić"> <!ENTITY mesto "Beograd"> <!ENTITY datum "16.7.2019."> ]> <pismo> <za>Saru Sarić</za> <naslov>Pozivnica</naslov> <poruka>Pozivam te na svoj rodjendan, u utorak 13.8.2019.</poruka> <futer> &autor;, &mesto; &datum;</futer> </pismo> Ovaj XML dokument će biti prikazan na sledeći način:

Entitet ima tri obavezna svoja dela, a to su ampersand (&), naziv entiteta i znak "tačka-zarez" (;).

Kada koristiti DTD a kada ne?

Zgodno je koristiti DTD u slučaju kada grupa ljudi radi na zajedničkom projektu, jer mogu standardizovati DTD za podatke koje svi koriste. Takodje, zahvaljujući DTD-u, uvek možete proveriti validnost podataka koje dobijate "sa strane" ili pokazati validnost vaših podataka.
XML DTD ne treba koristiti ukoliko pravite male XML dokumente jer, u tom slučaju, kreiranje DTD-a može biti samo gubljenje vremena, s obzirom na to da DTD nije obavezan u XML-u.
