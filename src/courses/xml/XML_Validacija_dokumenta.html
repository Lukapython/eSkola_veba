Validacija XML dokumenta

Za XML dokument kažemo da je dobro oformljen ako ima ispravnu sintaksu, odnosno ako su ispoštovana sintaksna pravila:
  • XML dokument mora imati koreni element.
  • XML elementi moraju imati završnu etiketu.
  • XML dokumenti su case sensitive (osetljivi na veličinu slova).
  • XML elementi moraju biti dobro ugnježdeni.
  • Vrednosti XML atributa moraju stajati pod navodnicima.

Bilo kakva greška u XML dokumentu će zaustaviti XML aplikaciju. Dakle, za razliku od HTML-a, gde pretraživači mogu da otvore HTML dokument sa greškom, u XML-u prostora za greške nema. Svaka aplikacija koja manipuliše XML dokumentom će biti zaustavljena ukoliko u njemu postoji greška. Razlog tome je to što bi XML dokument trebalo da bude mali, brz i kompatibilan.

Valjan (validan) XML dokument nije isto što i dobro oformljen XML dokument. Valjan XML dokument mora biti dobro oformljen, ali pored toga mora odgovarati definiciji tipa dokumenta. Postoje dve vrste definicija tipa dokumenata koje se primenjuju u XML-u:
  • DTD - originalno definisanje tipa dokumenata (Document type definition).
  • XML shema - alternativa DTD-u.
Definicija tipa dokumenata definiše pravila i ispravne elemente i atribute za XML dokument.
